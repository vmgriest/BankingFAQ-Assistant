# Bank FAQ Chatbot (Synthetic Demo)

A conversational banking FAQ assistant: FastAPI backend, TF-IDF retrieval over a synthetic FAQ
knowledge base, answer generation grounded via a **local Ollama LLM** (no cloud API key needed),
and a bubble-style multi-turn chat UI with an admin panel for managing the knowledge base.

> All data (rates, fees, policies) is fictional and for demo purposes only.

## Features

- Multi-turn chat with session memory (per-browser session id stored in `localStorage`)
- Handles phrasing/intent variation ("What's my savings rate?" vs "How much interest do I earn?")
  via TF-IDF retrieval + LLM paraphrase-tolerant generation
- Graceful fallback + contact-support suggestion for out-of-scope questions
- Confidence score + source section shown under each answer
- Suggested follow-up questions (LLM-generated) as clickable chips
- Frustration/sentiment detection (keyword + punctuation heuristics) with an on-page banner and a
  softened, escalation-aware reply
- Spanish support: auto language detection, or force via the language selector
- Admin panel (`/admin.html`) to add/edit/delete FAQ entries, persisted to `backend/data/faq.json`

## Prerequisites

- Python 3.10+
- [Ollama](https://ollama.com) installed and running locally
- A pulled model (default `llama3.2:3b`):
  ```bash
  ollama pull llama3.2:3b
  ```

## Setup & Run

```bash
# from the project root
python -m venv .venv
./.venv/Scripts/activate        # Windows
pip install -r backend/requirements.txt

# make sure Ollama is running in another terminal:
ollama serve

# start the app
uvicorn backend.main:app --reload --port 8000
```

Then open **http://localhost:8000** for the chat UI, or **http://localhost:8000/admin.html** for
the admin panel.

Optional environment variables:
- `OLLAMA_URL` (default `http://localhost:11434`)
- `OLLAMA_MODEL` (default `llama3.2:3b`)

## Project Layout

```

  main.py            FastAPI app: /api/chat, /api/faqs (CRUD), static frontend mount
  retrieval.py        TF-IDF FAQ search + FAQ CRUD persistence
  llm.py               Ollama chat calls, grounded prompt, follow-up generation, language detection
  sentiment.py        Frustration/sentiment heuristics
  data/
    faq.json            Synthetic FAQ knowledge base (23 entries across 5 categories)
    policy.md           Mock banking policy document used for grounding context
    sample_queries.json 20 sample customer queries with expected-answer notes
frontend/
  index.html, style.css, app.js     Chat UI
  admin.html, admin.css, admin.js   FAQ admin panel
```

## How answers are grounded

1. The user message is matched against `faq.json` via TF-IDF cosine similarity.
2. If the best match scores below a threshold, the request is treated as **out-of-scope** and a
   canned fallback (with contact-support info) is returned — no LLM call, so it can't hallucinate.
3. Otherwise, the top matching FAQ entries are passed to the local Ollama model as context, with
   an instruction to answer only from that context, in the requested language.
4. Follow-up questions are generated by asking the same model, given the conversation so far, for
   3 short natural next questions.

## Testing it

Try queries from `backend/data/sample_queries.json`, e.g.:
- "What's my savings rate?" vs "How much interest do I earn?" — same intent, different phrasing
- "Cuál es la tasa de interés de mi cuenta de ahorros?" — Spanish, auto-detected
- "This is ridiculous, nobody helps me!!" — triggers the frustration banner
- "Can you help me file my taxes?" — out-of-scope fallback
